# ClinicFlow AI — i18n Design Spec

**Date:** 2026-04-22
**Languages:** Hebrew (default, RTL), English (LTR), Russian (LTR)
**Scope:** Full app — landing page, auth pages, dashboard, all internal pages

---

## Architecture

**Library:** `next-intl` — industry standard for Next.js 14 App Router i18n.

**Routing:** URL-based locale prefix.
- `/` → redirects to `/he`
- `/he/dashboard`, `/en/dashboard`, `/ru/dashboard`
- `/he/leads`, `/en/leads`, `/ru/leads`
- etc.

**Default locale:** `he` (Hebrew). No auto-detection from browser.

**Direction:** `<html dir="rtl">` for `he`, `<html dir="ltr">` for `en` and `ru`.

---

### New files
| File | Purpose |
|---|---|
| `messages/he.json` | Hebrew translations (source of truth) |
| `messages/en.json` | English translations |
| `messages/ru.json` | Russian translations |
| `i18n.ts` | next-intl config — supported locales, default locale |
| `middleware.ts` | Updated — locale routing + existing auth protection |
| `components/layout/LanguageSwitcher.tsx` | Dropdown: עב / EN / RU |

### Restructured directories
| Before | After |
|---|---|
| `app/(app)/` | `app/[locale]/(app)/` |
| `app/(auth)/` | `app/[locale]/(auth)/` |
| `app/page.tsx` | `app/[locale]/page.tsx` |
| `app/layout.tsx` | `app/[locale]/layout.tsx` (locale-aware) + root `app/layout.tsx` (redirect only) |

## Language Switcher

Placed in two locations:
1. **Sidebar** (authenticated users) — below nav items, above bottom padding
2. **Landing page nav** — next to login/register buttons

UI: Three text buttons `עב | EN | RU` — active locale highlighted. Clicking navigates to same path with new locale prefix.

---

## Middleware

Combines locale routing with existing auth protection:

```typescript
// 1. Detect locale from URL prefix
// 2. If no locale prefix → redirect to /he/...
// 3. If accessing protected route without session → redirect to /he/login
// 4. If accessing auth route with session → redirect to /he/dashboard
```

---

## Layout Changes

`app/[locale]/layout.tsx`:
- Reads `locale` param
- Sets `<html lang={locale} dir={locale === "he" ? "rtl" : "ltr"}>`
- Wraps with `NextIntlClientProvider`

---

## What Does NOT Change

- Prisma schema, API routes, data layer — untouched
- Component logic — only string literals replaced with `t("key")` calls
- Design, colors, layout — unchanged
- Auth flow (NextAuth) — unchanged
